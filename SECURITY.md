# Security Policy

## Supported versions

SentinelBench is under active early development. Security fixes will target the latest `main` branch.

## Reporting a vulnerability

Do **not** open a public GitHub issue for security vulnerabilities.

Please report suspected vulnerabilities privately (for example via GitHub Security Advisories once enabled, or by contacting the maintainers through the channel listed in the repository).

Include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment, if known
- Any suggested remediation

We aim to acknowledge reports promptly and will coordinate disclosure after a fix is available.

## Dataset and content policy

- Do not upload exploit payloads, malware, or real confidential incident data.
- Only approved public datasets may be added under `data/` (see [data/README.md](data/README.md)).
- Threat-model and methodology docs describe evaluation intent; they are not attack how-tos.
